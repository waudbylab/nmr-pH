Documentation

This application estimates sample pH from NMR buffer chemical shifts using nonlinear least-squares fitting. It supports multiple buffers, nuclei, and can refine temperature, ionic strength, and reference offsets.

Theory and Methods

pH-Dependent Chemical Shifts

The observed chemical shift of a buffer resonance depends on the population-weighted average of the limiting chemical shifts for each ionisation state:

$$\delta_{\mathrm{obs}} = \sum_i f_i \cdot \delta_i \tag{1}$$

where \(f_i\) is the fractional population of ionisation state \(i\), and \(\delta_i\) is the limiting chemical shift for that state.

For a buffer with a single pKa, the fractional populations are given by the Henderson-Hasselbalch equation:

$$f_{\mathrm{protonated}} = \frac{1}{1 + 10^{\mathrm{pH} - \mathrm{p}K_a}} \tag{2}$$
$$f_{\mathrm{deprotonated}} = \frac{1}{1 + 10^{\mathrm{p}K_a - \mathrm{pH}}} \tag{3}$$

Temperature Dependence

The pKa varies with temperature according to the van't Hoff equation, with an optional heat capacity correction:

$$\mathrm{p}K_a(T) = \mathrm{p}K_a(T_{\mathrm{ref}}) + \frac{\Delta H}{R}\left(\frac{1}{T} - \frac{1}{T_{\mathrm{ref}}}\right) + \frac{\Delta C_p}{R}\left(\frac{T_{\mathrm{ref}}}{T} - 1 + \ln\frac{T}{T_{\mathrm{ref}}}\right) \tag{4}$$

where \(R = 8.314\) J/(mol·K) is the gas constant, \(\Delta H\) is the enthalpy of ionization, and \(\Delta C_p\) is the heat capacity change.

Chemical shifts also have temperature dependence, described by linear temperature coefficients:

$$\delta(T) = \delta(T_{\mathrm{ref}}) + \alpha_T \cdot (T - T_{\mathrm{ref}}) \tag{5}$$

where \(\alpha_T\) is the temperature coefficient in ppm/K.

Ionic Strength Dependence

The pKa is corrected for ionic strength using the Davies equation:

$$\mathrm{p}K_a(I) = \mathrm{p}K_a(I=0) + A \cdot \Delta z^2 \cdot \left(\frac{\sqrt{I}}{1+\sqrt{I}} - 0.3I\right) \tag{6}$$

where \(A \approx 0.51\) M-1/2 at 25°C, \(\Delta z^2\) is the change in squared charge upon deprotonation, and \(I\) is the ionic strength in molarity.

The ionic strength is calculated from the molar concentrations and charges of all ions in solution:

$$I = \frac{1}{2} \sum_i c_i \cdot z_i^2 \tag{7}$$

Chemical shifts may also have ionic strength dependence, described by linear coefficients:

$$\delta(I) = \delta(I=0) + \alpha_I \cdot I \tag{8}$$

where \(\alpha_I\) is the ionic strength coefficient in ppm/M.

Fitting Method

The application uses nonlinear least-squares fitting (Levenberg-Marquardt algorithm) to simultaneously determine pH from all observed chemical shifts. The fitting can optionally refine temperature, ionic strength, and/or chemical shift reference offsets.

Peak assignments are made automatically by matching observed shifts to the closest predicted resonance. Uncertainties in fitted parameters are estimated from the Jacobian matrix at the optimal solution.

Database Schema

The buffer database is a JSON file containing two main arrays: samples (experimental metadata) and buffers (buffer parameters).

Top-Level Structure

{
  "database_version": "0.0.1",
  "last_updated": "2025-01-01",
  "samples": [...],
  "buffers": [...]
}

Sample Object

Each sample describes the experimental conditions under which buffer data was measured:

FieldTypeDescription
sample_idstringUnique identifier (format: author_year_description_solvent)
solventenumOne of: "10pct_D2O", "100pct_D2O", "H2O", "other"
authorsarrayList of {name, orcid, affiliation}
reference_temperature_KnumberReference temperature in Kelvin (typically 298.15 K)
reference_ionic_strength_MnumberReference ionic strength in M
measurement_rangesobjectValid ranges for pH, temperature, ionic strength
temperature_calibrationstringMethod used for temperature calibration
pH_calibrationstringElectrode type and calibration standards used
ionic_strength_controlstringHow ionic strength was controlled
raw_data_repositoryobjectLink to raw NMR data (URL, commit SHA, format)

Buffer Object

Each buffer contains thermodynamic and NMR parameters:

FieldTypeDescription
buffer_idstringUnique identifier (format: author_year_name_solvent)
buffer_namestringHuman-readable name
buffer_familystringChemical family (e.g., "phosphate", "amine")
sample_idstringReference to parent sample
ionisation_statesintegerNumber of ionisation states (N+1 for N pKa values)
pKa_parametersarraypKa and thermodynamic data (one entry per pKa)
chemical_shiftsobjectChemical shift data organized by nucleus

Chemical Shift Deposition

Chemical shift information is stored for each nucleus with detailed metadata and coefficients describing the environmental dependencies. Each resonance entry contains:

FieldTypeDescription
resonance_idstringUnique identifier for the resonance (e.g., "H_alpha", "F_ortho")
descriptionstringChemical description of the resonating group
multiplicitystringPeak splitting pattern: "singlet", "doublet", "triplet", etc.
limiting_shiftsarrayArray of shift values for each ionisation state

Each entry in limiting_shifts specifies the chemical shift for one ionisation state:

FieldTypeDescription
ionisation_stateintegerIndex (0 = most protonated, N = fully deprotonated)
shift_ppm[value, unc]Chemical shift in ppm at reference conditions
temperature_coefficient_ppm_per_K[value, unc]Linear temperature coefficient (ppm/K)
ionic_strength_coefficient_ppm_per_M[value, unc]Linear ionic strength coefficient (ppm/M)
Uncertainty Format: All values with uncertainties are represented as [value, uncertainty] arrays. If uncertainty is unknown, use a single number. Temperature and ionic strength coefficients are optional and may be omitted if not measured.

Worked Example: Adding a New Buffer

This example demonstrates how to add a new buffer (imidazole) with 1H and 13C NMR data to the database. We'll create both a sample entry and a buffer entry.

Step 1: Create a Sample Entry

First, define the experimental conditions under which measurements were performed:

{
  "sample_id": "smith_2025_imidazole_100pctD2O",
  "solvent": "100pct_D2O",
  "authors": [
    {
      "name": "Jane Smith",
      "orcid": "0000-0001-2345-6789",
      "affiliation": "Example University"
    }
  ],
  "date_measured": "2025-01-15",

  "reference_temperature_K": 298.15,
  "reference_ionic_strength_M": 0.0,

  "measurement_ranges": {
    "pH": {"min": 5.0, "max": 9.0},
    "temperature_K": {"min": 288, "max": 308},
    "ionic_strength_M": {"min": 0.0, "max": 0.2}
  },

  "temperature_calibration": "Methanol chemical shift thermometer",
  "pH_calibration": "Combination pH electrode calibrated at pH 4.00, 7.00, 10.00",
  "ionic_strength_control": "KCl addition",

  "raw_data_repository": {
    "url": "https://github.com/example/data/imidazole_2025",
    "commit_sha": "a1b2c3d4",
    "format": "Bruker TopSpin format"
  }
}

Step 2: Create a Buffer Entry

Imidazole has a single pKa (~7.0), resulting in 2 ionisation states. We'll include 1H shifts for the C2, C4, and C5 protons, plus 13C shifts.

Step 3: Validate the Entry

Before adding to the database, verify that:

  • All IDs are unique and follow the naming convention
  • The number of ionisation states matches the pKa parameters (N pKa values → N+1 states)
  • Each resonance has shifts defined for all ionisation states
  • Uncertainties are provided where known
  • Temperature/ionic strength coefficients are included when measured
  • Peak multiplicity is specified for assignment guidance
Important: Always validate your JSON against the schema (draft-schema.json) before submitting. Ensure raw NMR data is deposited in a permanent repository with proper version control.
